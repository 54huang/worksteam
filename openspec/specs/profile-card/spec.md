# Profile Card

## Purpose

个人简介卡片组件，在页面正中央展示个人头像、邮箱（点击复制到剪贴板）、GitHub 链接及个人简介文字，适配桌面端和移动端。

## Requirements

### Requirement: 页面居中展示简介卡片

系统 SHALL 在页面视口正中显示一个圆角矩形简介卡片，卡片在水平和垂直方向均居中。

#### Scenario: 页面加载后卡片居中

- **WHEN** 用户打开页面
- **THEN** 页面正中央显示一个圆角矩形卡片，水平和垂直方向均居中

#### Scenario: 窗口大小改变时卡片保持居中

- **WHEN** 用户调整浏览器窗口大小
- **THEN** 卡片始终保持水平和垂直居中

### Requirement: 卡片展示个人头像

系统 SHALL 在卡片顶部显示圆形个人头像图片。

#### Scenario: 头像正常显示

- **WHEN** 页面加载完成
- **THEN** 卡片顶部显示圆形裁剪的个人头像

#### Scenario: 头像加载失败时有占位

- **WHEN** 头像图片无法加载
- **THEN** 显示默认占位图标或文字替代

### Requirement: 卡片展示邮箱信息并支持点击复制

系统 SHALL 在卡片中显示邮箱地址，用户点击邮箱后 SHALL 将邮箱地址复制到系统剪贴板。

#### Scenario: 点击邮箱复制到剪贴板

- **WHEN** 用户点击邮箱文字
- **THEN** 邮箱地址被复制到系统剪贴板

#### Scenario: 复制成功后给出反馈

- **WHEN** 邮箱地址成功复制到剪贴板
- **THEN** 页面显示"已复制"的短暂提示

### Requirement: 卡片展示 GitHub 链接

系统 SHALL 在卡片中提供 GitHub 图标链接，点击后在新标签页打开用户的 GitHub 主页。

#### Scenario: 点击 GitHub 图标跳转

- **WHEN** 用户点击 GitHub 图标
- **THEN** 在新标签页打开 GitHub 个人主页

### Requirement: 卡片展示个人简介文字

系统 SHALL 在卡片中显示一段个人简介文字，介绍基本信息。

#### Scenario: 个人简介文字正常显示

- **WHEN** 页面加载完成
- **THEN** 卡片中显示个人简介描述文字

### Requirement: 响应式布局适配

系统 SHALL 在不同屏幕尺寸下正常显示，卡片宽度在移动端不超过屏幕宽度。

#### Scenario: 移动端适配

- **WHEN** 用户在宽度小于 480px 的设备上打开页面
- **THEN** 卡片宽度自适应，左右留有适当边距，所有内容正常显示不溢出
