# Profile Card

## Purpose

个人简介卡片组件，在页面正中央展示个人头像、姓名、职位、技能标签、邮箱（点击复制到剪贴板）、GitHub 链接及个人简介短句，适配桌面端左右布局和移动端居中堆叠布局。

## Requirements

### Requirement: 页面居中展示简介卡片

系统 SHALL 在页面视口正中显示一个圆角矩形简介卡片，卡片在水平和垂直方向均居中。桌面端卡片内容采用左右布局（头像在左、姓名/职位在右），移动端采用居中堆叠布局。

#### Scenario: 页面加载后卡片居中

- **WHEN** 用户打开页面
- **THEN** 页面正中央显示一个圆角矩形卡片，水平和垂直方向均居中

#### Scenario: 窗口大小改变时卡片保持居中

- **WHEN** 用户调整浏览器窗口大小
- **THEN** 卡片始终保持水平和垂直居中

#### Scenario: 桌面端左右布局

- **WHEN** 用户在宽度大于等于 480px 的设备上打开页面
- **THEN** 头像左对齐，姓名和职位在头像右侧垂直排列

#### Scenario: 移动端堆叠布局

- **WHEN** 用户在宽度小于 480px 的设备上打开页面
- **THEN** 卡片内容切换为居中堆叠布局，头像居中、姓名/职位居中

### Requirement: 卡片展示个人头像

系统 SHALL 在卡片中显示圆形个人头像图片，桌面端左对齐，移动端居中显示。

#### Scenario: 头像正常显示

- **WHEN** 页面加载完成且头像图片可正常加载
- **THEN** 卡片中显示圆形裁剪的个人头像

#### Scenario: 头像加载失败时有占位

- **WHEN** 头像图片无法加载
- **THEN** 显示默认 SVG 占位图替代

### Requirement: 卡片展示个人职位

系统 SHALL 在姓名下方以灰色小字显示个人职位信息。

#### Scenario: 职位正常显示

- **WHEN** 页面加载完成
- **THEN** 姓名下方显示灰色职位文字（如"前端工程师"）

#### Scenario: 桌面端职位在头像右侧

- **WHEN** 用户在桌面端打开页面
- **THEN** 职位文字显示在姓名正下方，位于头像右侧区域

#### Scenario: 移动端职位居中

- **WHEN** 用户在移动端打开页面
- **THEN** 职位文字居中显示在姓名下方

### Requirement: 卡片展示个人简介短句

系统 SHALL 在卡片中显示一句简短的个人介绍 Tagline。

#### Scenario: Tagline 正常显示

- **WHEN** 页面加载完成
- **THEN** 卡片中显示简短的个人介绍文字（如"写代码十年，热爱开源"），位于姓名/职位下方、技能标签上方

### Requirement: 卡片展示技能标签

系统 SHALL 在卡片中展示技能标签，最多 8 个胶囊形标签，标签数据由 JavaScript 数组动态渲染。

#### Scenario: 技能标签正常显示

- **WHEN** 页面加载完成
- **THEN** 卡片中显示胶囊形技能标签，每行自动换行

#### Scenario: 标签数量不超过 8 个

- **WHEN** JavaScript 数组中技能超过 8 个
- **THEN** 仅渲染前 8 个标签

#### Scenario: 移动端标签自动换行

- **WHEN** 用户在移动端打开页面且标签超出卡片宽度
- **THEN** 标签自动换行显示，不溢出卡片

#### Scenario: JS 加载失败时技能标签区域为空但不阻塞页面

- **WHEN** 浏览器禁用 JavaScript 或脚本文件加载失败
- **THEN** 技能标签区域留空，卡片其余内容（头像、姓名、职位、Tagline、联系信息）仍正常显示

### Requirement: 卡片底部展示联系信息

系统 SHALL 在卡片底部用分割线隔开后显示邮箱和 GitHub 链接，桌面端左右分布，移动端居中堆叠。

#### Scenario: 桌面端联系信息左右分布

- **WHEN** 用户在桌面端打开页面
- **THEN** 邮箱在分割线下方左侧，GitHub 链接在右侧

#### Scenario: 移动端联系信息居中堆叠

- **WHEN** 用户在移动端打开页面
- **THEN** 邮箱和 GitHub 链接居中垂直排列

#### Scenario: 分割线可见

- **WHEN** 页面加载完成
- **THEN** 技能标签区域与联系信息区域之间显示一条水平分割线

### Requirement: 卡片展示邮箱信息并支持点击复制

系统 SHALL 在卡片中以圆角矩形容器展示邮箱地址，容器左侧显示信封图标，右侧显示邮箱文字。用户点击容器后 SHALL 将邮箱地址复制到系统剪贴板，并在页面顶部显示 toast 提示"已复制"，toast 在 1 秒内自动消失。

#### Scenario: 邮箱显示为圆角矩形带图标

- **WHEN** 页面加载完成
- **THEN** 邮箱地址以圆角矩形容器展示，左侧显示信封 SVG 图标，右侧显示邮箱文字

#### Scenario: 点击邮箱复制到剪贴板

- **WHEN** 用户点击邮箱容器
- **THEN** 邮箱地址被复制到系统剪贴板

#### Scenario: 复制成功后 1 秒内 toast 消失

- **WHEN** 邮箱地址成功复制到剪贴板
- **THEN** 页面顶部显示"已复制"toast，toast 在 1 秒内自动淡出消失

### Requirement: 卡片展示 GitHub 链接

系统 SHALL 在卡片底线提供 GitHub 图标链接，点击后在新标签页打开用户的 GitHub 主页。

#### Scenario: 点击 GitHub 图标跳转

- **WHEN** 用户点击 GitHub 图标按钮
- **THEN** 在新标签页打开 GitHub 个人主页

### Requirement: 响应式布局适配

系统 SHALL 在不同屏幕尺寸下正常显示，卡片宽度在移动端不超过屏幕宽度，布局在桌面端和移动端之间自动切换。

#### Scenario: 移动端适配

- **WHEN** 用户在宽度小于 480px 的设备上打开页面
- **THEN** 卡片宽度自适应，左右留有适当边距，布局自动切换为居中堆叠模式，所有内容正常显示不溢出
